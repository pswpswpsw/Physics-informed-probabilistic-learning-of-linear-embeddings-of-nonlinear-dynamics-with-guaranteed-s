MODEL_SRC.ClassDLKoopmanCont module¶

Class of Continuous time Deep Learning Models for Koopman Operator

class MODEL_SRC.ClassDLKoopmanCont.ClassDLKoopmanCont(configDict, edward_configDict=None, gpu_percentage=0.05)[source]¶

Bases: object

Class of a deterministic Deep learning model to find Koopman operators in a continuous time sense

For more details, please refer to preprint .

Args:

configDict (dict): a dictionary contains model options of deterministic DL for continuous Koopman operator.

Examples:

configDict = {
    'caseName': '50d_cylinder_flow_pod_case_noise_level_'+str(noise_level),
    "lr": 1e-4,
    "structureEncoder": [50, 100, 50, 20, 20],
    'numberEpoch': 30000,
    'miniBatch': 128,
    'decay': 1e-8,
    'c_los_lin': 1.0,
    'c_los_reg': 1e-10,
    'typeRecon': 'nonlinear',
    'phase_space_range': range_of_X,
    'activation': 'swish',
    'nsd': 1, # stable-K form.  #2: X-X^T -diag type  1:  efficient banded matrix - diag type  otherwise: not using anything
    'normalization_type':'only_max',
    'T': 100,
    'dt': 0.1*2 # 2 is the coarse-graining factor
}
edward_configDict (dict): a dictionary contains model options

of bayesian DL.

Examples:

edward_configDict = {
    'ALPHA_HPP': 0,
    'BETA_HPP': 1,
    'sm2g': {'pi': 1.0 / 4.0,
             's1': 10 ** 0,
             's2': 10 ** -6},
    'mode': 'ADVIARD',  # 'ADVIARD', 'MAPNoARD', 'MAPSimple', 'ADVInoARD',
    'init': 'Standard',# 'nonBayes', # 'Standard'
    'init_std_softplus': -8,
    'MAP_dir': '../result/50d_cylinder_flow_pod_case_noise_level_0/dldmd_2019-04-02-17-07-31',
    'verbose': False
}

gpu_percentage (float): ratio for how much GPU memory to use.

Attributes:
model_params (dict): the dictionary that contains all deterministic DL setup.
Besides, it adds two more as enable_edward, and edward_cfg to indicate using edward or not.
model_folder_name (str): name of the folder to put the saved model parameters. It is created
by current machine time, starting with dldmd_. If edward is enabled, it will add _Bayes as suffix.

dir (str): the relative path to put the model into.

initializer (function): an initializer (it is still a function).

scale_initializer (function): an initializer for scale in variational posteriori.

sess (class): A context manager using this session as the default session

graph (class): A Graph instance supports an arbitrary number of “collections”
that are identified by name.

vp_dict (dict): a dict collection of variational posteriori distributions.

hpp_dict (dict): a dict collection of hyperprior distribution.

prior_dict (dict): a dict collection of priori distribution.

K_X (tf.Tensor): Prob. distribution to construct stab. form.

K_XX (tf.Tensor): anti-symmetric pro. distribution to construct stab. form.

square_diagonal_line (tf.Tensor): the non-negative diagonal elements to construct stab. form.
It needs to be act with tf.diag to become really a matrix.

koopmanOp_learn_intermediate (tf.Tensor): the construct tensor of Koopman operator K.

koopmanOp_learn (tf.Tensor): the final tensor with name K of Koopman operator K.

pi (float): Scale Mixture 2 Gaussian prior hyperparameter: \(\pi\) in Google 2015 BNN paper: `at
<http://arxiv.org/abs/1505.05424>`_.
s1 (float): Scale Mixture 2 Gaussian prior hyperparameter: \(s_1\) in Google 2015 BNN paper: `at
<http://arxiv.org/abs/1505.05424>`_.
s2 (float): Scale Mixture 2 Gaussian prior hyperparameter: \(s_2\) in Google 2015 BNN paper: `at
<http://arxiv.org/abs/1505.05424>`_.

dimX (int): dimension of the input, i.e., system state \(x\).

numKoopmanModes (int): dimension of the Koopman invariant observable space.

activation (function): nonlinear activations in neural network.

shape_list (list): [numKoopmanModes,:attr:numKoopmanModes]. This is the shape of Koopman operator K.

encoderLayerWeights (dict): a dict collection of weights and biases for encoder.

decoderLayerWegihts (dict): a dict collection of weights and biases for decoder.

X (tf.Tensor): placeholder for the input states with shape as (None, dimX).

eta (tf.Tensor): normalized input \(\eta = (X - \overline{X}) \Lambda^{-1}\),
where \(\overline{X}\) = mu_X, \(\Lambda^{-1}\) = Lambda_X_inv.

Xdot (tf.Tensor): placeholder for the input of time derivative as \(\dot{X}\).

etaDot (tf.Tensor): normalized time derivative as \(\dot{\eta} = \dot{X} \Lambda^{-1}\).

z_rec_loss (tf.Tensor): placeholder for reconstruction loss for Edward to build likelihood function.

z_lin_loss (tf.Tensor): placeholder for linear dynamics loss for Edward to build likelihood function.

koopmanPhi (tf.Tensor): Koopman observables.

etaRec (tf.Tensor): reconstructed normalized states.

Xrec (tf.Tensor): reconstructed states as \(\overline{X} + \eta \Lambda\).

rec_loss (tf.Tensor): mean square error between X and Xrec.

norm_rec_loss (tf.Tensor): MSE between eta and etaRec.

fdphidx (tf.Tensor): \(\dot{\eta} \cdot \partial \phi /\partial X\).

kphi (tf.Tensor): \(\phi K\).

lin_loss (tf.Tensor): MSE between kphi and fdphidx.

reg_parameter_loss (tf.Tensor): regularization parameter loss with biases and weights.

loss_op (tf.Tensor): total loss objective function.

optimizer (class): optimizer class instance.

train_op (function) a function that updates variables in var_list,. i.e., the backpropagation
algorithm.

Xtrain (numpy.ndarray): training states \(X\).

XdotTrain (numpy.ndarray): training time derivative \(\dot{X}\).

Xvalid (numpy.ndarray): valid states \(X\).

XdotValid (numpy.ndarray): valid time derivative \(\dot{X}\).

mu_X (tf.Tensor): sampled mean of training states.

Lambda_X (tf.Tensor): sampled mean of training time derivative.

loss_dict (dict): a collection of training loss to be saved.

K_sample (numpy.ndarray): array of Monte Carlof sampling of K matrix.
